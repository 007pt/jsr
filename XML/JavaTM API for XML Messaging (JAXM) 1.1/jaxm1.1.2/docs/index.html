JAXM 1.1.2 Reference Implementation 

The JAXM 1.1.2 Reference Implementation (RI) is an implementation of an XML-based messaging system. It is based on the following specification:
  • JavaTM API for XML Messaging (JAXM) 1.1
    This specification covers the javax.xml.messaging package, which provides the API for using a messaging provider to send messages.
Note: SOAP with Attachments API for JavaTM (SAAJ) specification covers the javax.xml.soap package, which provides the API for creating and sending SOAP messages. This release of JAXM works with SAAJ 1.1 and above versions.

The overall goal is to provide a lightweight implementation to help build business-to-business (b-to-b) applications using JAXM. Typical b-to-b scenarios generally involve two or more business entities communicating via the Internet using a common protocol, such as HTTP.

This implementation tries to provide an application programming model for servlet-based b-to-b applications.

The rest of this document briefly describes what is in this release.
 

The Core

This client-side runtime library can be used to send messages to remote parties using a remote provider. Messages can be received asynchronously (using a one-way model) or synchronously (using a request-response model, which is a part of SAAJ). In this release, support for the asynchronous receipt of messages is implemented using a servlet.

JAXM 1.1: Communication Using a Remote Provider

A remote provider, also called a messaging provider, is akin to a messaging server. The messaging provider takes a message sent by an application and holds on to it until the message has been successfully delivered. The application itself may be active or quiescent, but the remote provider continues to try to deliver any messages that the application has sent and to receive any incoming messages on the application's behalf. When an application comes up and establishes a connection to the remote provider, messages received by the provider for that application are delivered to it. All messages sent through a remote provider are logged for perusal later. APIs to work with the remote provider are in the javax.xml.messaging package.

Provider Admin Tool


The JAXM provider administration tool allows users to make changes to the provider configuration. It is expected that the main users will be Systems Administrators, but anyone who needs to change the configuration for the provider can use it.

To start the tool, point your browser to http://localhost:8080/jaxm-provideradmin. Use the username and password that has role "provider" from tomcat-users.xml.

The provider war must be deployed in the container in the expanded mode and provider must be up and running before using the provider admin tool.

Samples

  • jaxm-simple - A simple example of sending and receiving messages using the local provider.
  • jaxm-remote - An example that uses a remote provider to send and receive messages using the ebXML profile.
  • JAXM tags - An example that uses JSP tags to generate and consume SOAP messages
  • jaxm-SOAPRP - jaxm-SOAPRP - An example of a roundtrip message exchange via a provider, using the SOAP-RP profile
